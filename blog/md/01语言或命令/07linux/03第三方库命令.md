### **类别一：网络请求与数据传输 (Network Request & Data Transfer)**

这类工具专注于通过网络获取或发送数据，是系统管理员和开发人员日常工作的基础。

#### **1. curl**
- **简介**: `curl` 是一个功能强大的命令行工具，用于通过 URL 语法传输数据。它支持众多协议，包括 HTTP, HTTPS, FTP, FTPS, SCP, SFTP, LDAP 等。它是网络调试、API 测试和文件下载的瑞士军刀。
- **主要用途**:
    - 下载文件。
    - 测试 RESTful API (发送 GET, POST, PUT, DELETE 等请求)。
    - 查看 HTTP 响应头。
    - 自动化脚本中的网络操作。
- **安装包名**:
    - **Debian/Ubuntu**: `curl`
      ```bash
      sudo apt update && sudo apt install curl
      ```
    - **RHEL/CentOS/Fedora**: `curl`
      ```bash
      sudo yum install curl  # 或 dnf install curl
      ```

#### **2. wget**
- **简介**: `wget` 是一个简单、非交互式的网络下载工具。它非常适合在脚本中或后台下载大型文件或整个网站。与 `curl` 相比，`wget` 更专注于“下载”这一核心功能。
- **主要用途**:
    - 从 HTTP/HTTPS/FTP 下载文件。
    - 递归下载，镜像整个网站或目录。
    - 在网络不稳定的情况下进行断点续传。
    - 后台下载。
- **安装包名**:
    - **Debian/Ubuntu**: `wget`
      ```bash
      sudo apt update && sudo apt install wget
      ```
    - **RHEL/CentOS/Fedora**: `wget`
      ```bash
      sudo yum install wget  # 或 dnf install wget
      ```

#### **3. rsync**
- **简介**: `rsync` (Remote Sync) 是一个用于文件同步和传输的高效工具。它通过“delta-transfer”算法，只传输源和目标之间有差异的部分，从而最大程度地减少数据传输量。
- **主要用途**:
    - 本地或远程文件/目录的备份与同步。
    - 网站部署和代码发布。
    - 增量备份。
- **安装包名**: (通常系统会预装)
    - **Debian/Ubuntu**: `rsync`
    - **RHEL/CentOS/Fedora**: `rsync`

### **类别二：网络分析与安全 (Network Analysis & Security)**

这类工具用于诊断网络问题、监控网络流量和进行安全审计。

#### **1. tcpdump**
- **简介**: `tcpdump` 是一个强大的命令行网络抓包工具。它允许用户捕获和分析流经网卡的 TCP/IP 和其他网络数据包。是网络故障排查的终极利器。
- **主要用途**:
    - 实时监控网络流量。
    - 捕获网络数据包以供后续分析（例如用 Wireshark 打开）。
    - 诊断网络连接问题，如“三次握手”是否成功。
    - 分析特定协议的通信内容。
- **安装包名**:
    - **Debian/Ubuntu**: `tcpdump`
    - **RHEL/CentOS/Fedora**: `tcpdump`

#### **2. Nmap**
- **简介**: `Nmap` (Network Mapper) 是一个开源的网络发现和安全审计工具。系统管理员用它来发现网络上的主机、开放的端口、运行的服务及其版本、操作系统等信息。
- **主要用途**:
    - 网络探测和主机发现。
    - 端口扫描，找出目标主机上开放的端口。
    - 服务和版本探测。
    - 操作系统探测。
    - 安全漏洞扫描（通过 NSE 脚本引擎）。
- **安装包名**:
    - **Debian/Ubuntu**: `nmap`
    - **RHEL/CentOS/Fedora**: `nmap`

#### **3. net-tools**
- **简介**: 这是一个**传统的**网络工具集，包含了如 `ifconfig`, `netstat`, `route` 等经典命令。虽然在现代 Linux 系统中，它正逐渐被 `iproute2` 工具包（如 `ip addr`, `ss`, `ip route`）取代，但由于历史悠久，仍被广泛使用。
- **主要用途**:
    - `ifconfig`: 配置和显示网络接口信息。
    - `netstat`: 显示网络连接、路由表、接口统计等。
    - `route`: 显示和操作 IP 路由表。
- **安装包名**: (在新系统中可能需要手动安装)
    - **Debian/Ubuntu**: `net-tools`
    - **RHEL/CentOS/Fedora**: `net-tools`

### **类别三：文本处理 (Text Processing)**

这些是 Linux Shell 编程的基石，用于解析、过滤和转换文本数据。

#### **1. sed**
- **简介**: `sed` (Stream Editor) 是一个流编辑器，用于对输入流（文件或管道输入）进行基本的文本转换。它以行为单位处理文本，功能强大且高效。
- **主要用途**:
    - 文本替换（查找并替换）。
    - 删除、插入、追加行。
    - 结合正则表达式进行复杂的文本匹配和处理。
- **安装包名**:
    - 通常是系统核心组件 (GNU sed)，**无需单独安装**。

#### **2. awk**
- **简介**: `awk` 是一种强大的文本处理语言。它擅长将文本文件按行读取，并按列（字段）进行处理。非常适合处理和分析结构化的文本数据，如日志文件、CSV 文件等。`gawk` 是其在 Linux 上的标准实现。
- **主要用途**:
    - 格式化和重新组织文本报告。
    - 从文本数据中提取特定列。
    - 对数据列进行计算（求和、平均等）。
    - 基于模式匹配执行特定动作。
- **安装包名**:
    - 通常是系统核心组件 (`gawk`)，**无需单独安装**。

#### **3. grep**
- **简介**: `grep` (Global Regular Expression Print) 是一个强大的文本搜索工具，它使用正则表达式搜索文本，并把匹配的行打印出来。
- **主要用途**:
    - 在文件中查找包含特定字符串或模式的行。
    - 从命令输出中过滤出需要的信息。
    - 统计匹配行的数量。
- **安装包名**:
    - 通常是系统核心组件，**无需单独安装**。

#### **4. jq**
- **简介**: `jq` 可以被看作是 "JSON 的 sed"。它是一个轻量级、灵活的命令行 JSON 处理器。在与返回 JSON 的 API 交互时，`jq` 是一个不可或缺的工具。
- **主要用途**:
    - 格式化和美化 JSON 输出。
    - 从复杂的 JSON 数据中提取特定字段的值。
    - 过滤和转换 JSON 数据结构。
- **安装包名**:
    - **Debian/Ubuntu**: `jq`
    - **RHEL/CentOS/Fedora**: `jq` (可能需要先启用 EPEL 源)


### **类别四：系统监控与性能分析 (System Monitoring & Performance Analysis)**

这类工具帮助管理员实时了解系统负载、资源瓶颈，并进行性能诊断。

#### **1. htop**
- **简介**: `htop` 是一个交互式的进程查看器，是传统 `top` 命令的增强版。它提供了更友好的用户界面，支持颜色显示、垂直和水平滚动、树状视图以及直接在界面中对进程进行操作（如 kill, renice）。
- **主要用途**:
  - 实时查看 CPU、内存、交换空间使用率。
  - 以树状视图查看进程间的父子关系。
  - 方便地搜索、过滤和管理进程。
- **安装包名**:
  - **Debian/Ubuntu**: `htop`
  - **RHEL/CentOS/Fedora**: `htop` (在较旧的 RHEL/CentOS 上可能需要 EPEL 源)

#### **2. iotop**
- **简介**: `iotop` 类似于 `top`，但它监控的是磁盘 I/O 使用情况。它按进程或线程显示实时的读写速率，对于定位导致磁盘性能瓶颈的进程非常有用。
- **主要用途**:
  - 查看哪个进程正在进行大量的磁盘读写。
  - 诊断磁盘 I/O 瓶颈。
  - 监控总的磁盘读写带宽。
- **安装包名**: (通常需要 root 权限运行)
  - **Debian/Ubuntu**: `iotop`
  - **RHEL/CentOS/Fedora**: `iotop`

#### **3. iftop**
- **简介**: `iftop` 类似于 `top`，但用于监控网络流量。它监听指定的网络接口，并以列表形式显示主机之间的带宽使用情况，帮助你快速找出网络流量的来源和去向。
- **主要用途**:
  - 实时监控网络接口的带宽消耗。
  - 找出占用网络带宽最多的连接和主机。
- **安装包名**: (通常需要 root 权限运行)
  - **Debian/Ubuntu**: `iftop`
  - **RHEL/CentOS/Fedora**: `iftop` (通常需要 EPEL 源)

#### **4. glances**
- **简介**: `glances` 是一个跨平台的、基于 Python 的全能系统监控工具。它在一个界面中整合了大量系统信息，包括 CPU, 内存, 负载, 进程列表, 网络接口, 磁盘 I/O, 文件系统, 传感器, 甚至 Docker 容器的统计信息。
- **主要用途**:
  - 提供系统资源的“一瞥式”概览。
  - 具备 Web UI 和 REST API，支持远程监控。
  - 可设置阈值并高亮显示警告或严重状态。
- **安装包名**:
  - **Debian/Ubuntu**: `glances`
  - **RHEL/CentOS/Fedora**: `glances` (通常需要 EPEL 源)

### **类别五：进程与服务管理 (Process & Service Management)**

除了系统自带的 `systemd` 或 `init.d`，某些第三方工具提供了更灵活的进程管理能力，尤其适用于应用程序层面。

#### **1. Supervisor**
- **简介**: `Supervisor` 是一个用 Python 编写的客户端/服务器系统，它允许用户在类 UNIX 系统上控制多个进程。它主要用于管理和监控需要长时间运行的程序，确保它们在意外退出时能自动重启。
- **主要用途**:
  - 守护和管理应用程序进程（如 Gunicorn, Celery, Node.js 服务）。
  - 进程崩溃时自动重启。
  - 将多个进程作为一个逻辑组进行管理（启动/停止/重启）。
  - 提供 `supervisorctl` 命令行工具和 Web 界面进行管理。
- **安装包名**:
  - **Debian/Ubuntu**: `supervisor`
  - **RHEL/CentOS/Fedora**: `supervisor` (通常需要 EPEL 源)
  - *也可以通过 Python 的包管理器安装*: `pip install supervisor`

### **类别六：文件归档与压缩 (File Archiving & Compression)**

尽管 `tar`, `gzip`, `bzip2` 等工具通常是系统自带的，但还有一些常用的工具用于处理不同的归档格式或提供额外的功能。

#### **1. tree**
- **简介**: `tree` 是一个非常实用的小工具，它以树状图的形式递归地列出目录的内容，使得目录结构一目了然。
- **主要用途**:
  - 可视化地展示目录和文件结构。
  - 可以显示文件大小、权限等信息。
- **安装包名**:
  - **Debian/Ubuntu**: `tree`
  - **RHEL/CentOS/Fedora**: `tree`

#### **2. zip / unzip**
- **简介**: 用于处理 `.zip` 格式的归档文件。`zip` 格式因其在 Windows 系统上的广泛使用而非常流行，是跨平台文件交换的常用选择。
- **主要用途**:
  - 创建 `.zip` 压缩包 (`zip`)。
  - 解压 `.zip` 压缩包 (`unzip`)。
- **安装包名**:
  - **Debian/Ubuntu**: `zip`, `unzip`
  - **RHEL/CentOS/Fedora**: `zip`, `unzip`

#### **3. p7zip**
- **简介**: `p7zip` 是著名的开源归档软件 7-Zip 的命令行版本。它支持多种归档格式，特别是其自有的 `.7z` 格式，通常能提供比 `zip` 或 `gzip` 更高的压缩率。
- **主要用途**:
  - 创建和解压高压缩率的 `.7z` 文件。
  - 处理多种其他格式，如 ZIP, GZIP, BZIP2, TAR, RAR 等。
- **安装包名**:
  - **Debian/Ubuntu**: `p7zip-full` (包含 7z, 7za, 7zr)
  - **RHEL/CentOS/Fedora**: `p7zip`, `p7zip-plugins`

### **类别八：容器化与虚拟化 (Containerization & Virtualization)**

#### **1. QEMU/KVM**
- **简介**: `KVM` (Kernel-based Virtual Machine) 是 Linux 内核的一个模块，它使 Linux 成为一个类型一（裸机）的虚拟机监控器。`QEMU` 是一个通用的开源机器模拟器和虚拟器。当与 KVM 结合使用时，QEMU 可以利用硬件虚拟化扩展（如 Intel VT-x 或 AMD-V）来运行虚拟机，从而获得接近本机的性能。
- **主要用途**:
  - 在 Linux 主机上创建和运行完整的虚拟机（如 Windows, 其他 Linux 发行版等）。
  - 提供 IaaS (基础设施即服务) 云环境的底层技术（如 OpenStack）。
  - 系统级隔离和测试。
- **安装包名**: (通常需要安装一组包)
  - **Debian/Ubuntu**: `qemu-kvm`, `libvirt-daemon-system`, `libvirt-clients`, `bridge-utils`, `virt-manager` (GUI 管理工具)
  - **RHEL/CentOS/Fedora**: `qemu-kvm`, `libvirt`, `virt-install`, `bridge-utils`, `virt-manager` (GUI 管理工具)

#### **2. fail2ban**
- **简介**: `fail2ban` 是一个入侵防御软件框架。它通过扫描日志文件（如 `/var/log/auth.log` 或 `/var/log/secure`）来检测恶意行为，如过多的密码错误、寻找漏洞等，并自动更新防火墙规则来阻止相应的 IP 地址。
- **主要用途**:
  - 保护 SSH 服务免受暴力破解攻击。
  - 保护 Web 服务器（如 Nginx, Apache）的登录页面。
  - 保护 FTP、邮件服务器等。
- **安装包名**:
  - **Debian/Ubuntu**: `fail2ban`
  - **RHEL/CentOS/Fedora**: `fail2ban` (通常需要 EPEL 源)


### **类别九：自动化与配置管理 (Automation & Configuration Management)**

这类工具能够将基础设施的配置代码化（Infrastructure as Code），实现服务器集群的自动化部署、配置和管理，极大地提高了效率和一致性。

#### **1. Ansible**
- **简介**: `Ansible` 是一个极其流行的开源自动化工具。它使用简单的 YAML 语言来描述自动化任务（称为 Playbooks）。其最大的特点是**无代理 (Agentless)**，通过 SSH 连接到受管节点并执行任务，因此部署和使用非常方便。
- **主要用途**:
  - 应用程序部署。
  - 配置管理（确保服务器配置的一致性）。
  - IT 流程编排（按顺序执行跨越多台服务器的复杂任务）。
  - 滚动更新和持续交付。
- **安装包名**:
  - **Debian/Ubuntu**: `ansible`
  - **RHEL/CentOS/Fedora**: `ansible` (在较新版本中是 `ansible-core`，通常需要 EPEL 源)
  - *也可以通过 Python 的包管理器安装*: `pip install ansible`

*注：此领域还有其他著名工具，如 `Puppet`, `Chef`, 和 `SaltStack`，它们通常采用客户端/服务器（Agent/Master）架构。*

### **类别十：日志管理与聚合 (Log Management & Aggregation)**

在拥有多台服务器或大量容器的环境中，将日志集中存储、处理和查询是进行故障排查、性能监控和安全审计的基础。

#### **1. The ELK Stack (Elastic Stack)**
- **简介**: `ELK` 是三个开源项目的首字母缩写，它们组合在一起构成一个强大的日志管理平台：
  - **E**lasticsearch: 一个分布式的搜索和分析引擎，用于存储和索引日志。
  - **L**ogstash: 一个服务器端的数据处理管道，能够接收来自多个来源的数据，转换数据，然后将其发送到像 Elasticsearch 这样的“存储库”中。
  - **K**ibana: 一个 Web UI，用于搜索、分析和可视化 Elasticsearch 中的数据（日志）。
- **主要用途**:
  - 集中化日志存储和管理。
  - 对海量日志数据进行快速的全文搜索。
  - 创建仪表盘（Dashboard）来可视化系统健康状况、业务指标等。
- **安装包名**: (强烈建议从 Elastic 官方仓库安装)
  - **Debian/Ubuntu**: `elasticsearch`, `logstash`, `kibana`
  - **RHEL/CentOS/Fedora**: `elasticsearch`, `logstash`, `kibana`

#### **2. Filebeat / Fluentd**
- **简介**: `Filebeat` 和 `Fluentd` 是轻量级的日志数据**收集和转发**工具（Shipper）。它们通常部署在产生日志的服务器上，负责监视日志文件，并将新的日志条目发送到 Logstash 或直接发送到 Elasticsearch。相比 Logstash，它们占用的资源更少。
- **主要用途**:
  - 从成百上千台服务器上高效地收集日志。
  - `Filebeat` 是 Elastic Stack 的一部分，与 ELK 集成最为紧密。
  - `Fluentd` 是 CNCF (云原生计算基金会) 的项目，拥有非常丰富的插件生态，可以连接到各种数据源和数据目的地。
- **安装包名**:
  - **Filebeat**:
    - **Debian/Ubuntu**: `filebeat` (从 Elastic 官方仓库)
    - **RHEL/CentOS/Fedora**: `filebeat` (从 Elastic 官方仓库)
  - **Fluentd**:
    - 通常通过其官方提供的 `td-agent` 包进行安装。

### **类别十一：文件传输服务 (File Transfer Services)**

这些工具提供了标准化的协议，用于在不同系统之间上传和下载文件。

#### **1. vsftpd (FTP Server)**
- **简介**: `vsftpd` (Very Secure FTP Daemon) 是一个在类 UNIX 系统上广泛使用的 FTP 服务器软件。它以安全、稳定和高性能为设计目标。FTP 是一个经典的文件传输协议。
- **主要用途**:
  - 搭建一个允许用户通过 FTP 客户端（如 FileZilla）上传和下载文件的服务器。
  - 支持 FTP、FTPS (FTP over SSL/TLS) 等模式。
- **安装包名**:
  - **Debian/Ubuntu**: `vsftpd`
  - **RHEL/CentOS/Fedora**: `vsftpd`

#### **2. OpenSSH Server (for SFTP)**
- **简介**: **SFTP (SSH File Transfer Protocol)** 是一个与 FTP 完全不同的、更安全的文件传输协议。它不使用 FTP 协议，而是作为 SSH (Secure Shell) 协议的一个子系统来运行。因此，任何运行了 SSH 服务器的主机，默认就具备了提供 SFTP 服务的能力。
- **主要用途**:
  - 提供一个加密的、安全的文件传输通道。
  - 利用现有的 SSH 用户认证体系进行权限控制。
  - 现代文件传输的首选，替代不安全的 FTP 和复杂的 FTPS。
- **安装包名**: (安装包是 SSH 服务器本身)
  - **Debian/Ubuntu**: `openssh-server` (通常默认已安装)
  - **RHEL/CentOS/Fedora**: `openssh-server` (通常默认已安装)


### **类别十二：安全审计与漏洞扫描 (Security Auditing & Vulnerability Scanning)**

除了前面提到的 `Nmap` 和 `fail2ban`，还有一些更专门的工具用于深度安全分析和系统加固。

#### **1. Lynis**
- **简介**: `Lynis` 是一个针对类 UNIX 系统的安全审计和系统加固工具。它会执行数百项独立的测试，以检查系统的安全状况，并提供修复建议。
- **主要用途**:
  - 安全审计和评估。
  - 合规性测试（如 ISO27001, PCI-DSS, HIPAA）。
  - 漏洞检测和系统加固建议。
- **安装包名**:
  - **Debian/Ubuntu**: `lynis`
  - **RHEL/CentOS/Fedora**: `lynis` (通常需要 EPEL 源)
  - *也可以直接从官网或 GitHub 克隆并运行。*

#### **2. ClamAV**
- **简介**: `ClamAV` 是一个开源的防病毒引擎，用于检测木马、病毒、恶意软件和其他恶意威胁。它在 Linux 世界中尤其常用于邮件网关和文件服务器的扫描。
- **主要用途**:
  - 扫描文件和目录。
  - 扫描邮件附件。
  - 实时保护（通过 `clamd` 守护进程）。
- **安装包名**: (通常需要安装主程序和守护进程)
  - **Debian/Ubuntu**: `clamav`, `clamav-daemon`
  - **RHEL/CentOS/Fedora**: `clamav`, `clamav-update`

#### **3. OpenVAS / GVM**
- **简介**: `Greenbone Vulnerability Management (GVM)`，之前被称为 `OpenVAS`，是一个功能全面的漏洞扫描和管理解决方案。它是著名商业扫描器 Nessus 的一个开源分支，功能非常强大。
- **主要用途**:
  - 全面的网络漏洞扫描。
  - 生成详细的安全报告。
  - 持续的漏洞生命周期管理。
- **安装包名**:
  - 安装过程**相对复杂**，通常涉及多个组件。强烈建议遵循其官方文档。
  - **Debian/Ubuntu**: 包名为 `gvm`。
  - **RHEL/CentOS/Fedora**: 通常需要从源码或第三方仓库编译安装。

### **类别十三：持续集成与部署 (Continuous Integration & Deployment - CI/CD)**

CI/CD 工具可以自动化软件的构建、测试和发布流程，是现代 DevOps 文化的核心。

#### **1. Jenkins**
- **简介**: `Jenkins` 是一个用 Java 编写的、开源的、可扩展的自动化服务器。它是 CI/CD 领域事实上的标准之一，拥有庞大的插件生态系统，可以与几乎所有开发、测试和部署工具集成。
- **主要用途**:
  - 自动化的代码构建（编译）。
  - 运行自动化测试。
  - 部署应用程序到测试或生产环境。
  - 创建复杂的 CI/CD 流水线 (Pipelines)。
- **安装包名**:
  - **强烈建议使用官方源进行安装**，因为它是一个独立的 Java 应用。
  - 首先需要安装 Java 环境 (`openjdk-11-jdk` 或更高版本)。
  - 然后添加 Jenkins 官方的 `apt` 或 `yum` 仓库，再安装 `jenkins` 包。

#### **2. GitLab CI/CD**
- **简介**: `GitLab CI/CD` 是深度集成在 GitLab 平台中的一个强大功能。如果你的代码托管在 GitLab 上，使用其自带的 CI/CD 是一个非常自然和高效的选择。它通过项目根目录下的一个 `.gitlab-ci.yml` 文件进行配置。
- **主要用途**:
  - 与 GitLab 代码仓库无缝集成的构建、测试、部署流程。
  - 支持 Docker，可以方便地在容器中运行 CI/CD 任务。
  - 提供 Auto DevOps 功能，能自动检测、构建、测试和部署应用。
- **安装包名**:
  - **它不是一个独立的软件**，而是 GitLab 平台（社区版 `gitlab-ce` 或企业版 `gitlab-ee`）的一部分。安装 GitLab 即获得了 GitLab CI/CD 的能力。你还需要配置和注册 `GitLab Runner` 来执行任务。

### **类别十四：终端效率与Shell增强 (Terminal Productivity & Shell Enhancement)**

这些工具能让你的命令行体验发生质的飞跃。

#### **1. tmux / screen**
- **简介**: `tmux` 和 `screen` 是终端复用器 (Terminal Multiplexer)。它们允许你在单个终端窗口中创建和管理多个独立的 Shell 会话，并且可以在断开 SSH 连接后保持这些会话的运行。`tmux` 通常被认为是 `screen` 的一个更现代、功能更丰富的替代品。
- **主要用途**:
  - 在一个终端里创建多个窗口和窗格（分屏）。
  - 安全地断开远程 SSH 连接，稍后再重新连接并恢复工作现场。
  - 结对编程 (Pair Programming)。
- **安装包名**:
  - **Debian/Ubuntu**: `tmux`, `screen`
  - **RHEL/CentOS/Fedora**: `tmux`, `screen`

#### **2. fzf**
- **简介**: `fzf` 是一个通用、极速的命令行模糊查找器 (Fuzzy Finder)。它可以与许多其他命令和 Shell 功能集成，彻底改变你查找文件、命令历史和进程的方式。
- **主要用途**:
  - 交互式地反向搜索命令历史 (`Ctrl+R`)。
  - 快速查找和打开文件 (`Ctrl+T`)。
  - 结合 `kill` 命令快速查找并终止进程。
- **安装包名**:
  - **Debian/Ubuntu**: `fzf`
  - **RHEL/CentOS/Fedora**: `fzf`
  - *为了获得最佳体验，建议运行其附带的安装脚本来完成 Shell 集成。*

#### **3. bat**
- **简介**: `bat` 是一个 `cat` 命令的克隆版本，但增加了许多现代化的功能，最主要的是语法高亮和 Git 集成。
- **主要用途**:
  - 以更美观的方式查看文件内容（带语法高亮和行号）。
  - 显示文件的 Git 修改状态。
  - 可以与 `fzf` 等工具集成。
- **安装包名**:
  - **Debian/Ubuntu**: `bat` (在一些较新版本中，为了避免与另一个工具冲突，包名可能为 `batcat`，需要创建一个 `bat` 的符号链接)
  - **RHEL/CentOS/Fedora**: `bat`

#### **4. zsh & Oh My Zsh**
- **简介**: `zsh` (Z Shell) 是一个功能比默认的 `bash` 更强大的 Shell。而 `Oh My Zsh` 是一个社区驱动的、用于管理 `zsh` 配置的框架。它使得使用 `zsh` 的高级功能（如插件、主题、自动补全）变得极其简单。
- **主要用途**:
  - 极其强大的自动补全功能（能补全命令、参数、文件名、Git 分支等）。
  - 通过插件扩展功能（如 `git`, `docker`, `sudo` 等插件）。
  - 使用丰富的主题美化你的终端提示符。
- **安装包名**:
  - **1. 安装 zsh**:
    - **Debian/Ubuntu**: `sudo apt install zsh`
    - **RHEL/CentOS/Fedora**: `sudo yum install zsh`
  - **2. 安装 Oh My Zsh**:
    - 通常通过官网提供的 `curl` 或 `wget` 一行命令脚本来安装。
